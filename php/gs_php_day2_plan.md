# Day2 授業設計（3時間版）

## 前提

- 受講生：HTML/CSS/JS を約10時間学んだ初学者
- 授業時間：50分 × 3コマ（計150分）
- 宿題：ブックマークアプリ（授業内では扱わない）
- AI活用：調べもの程度には使える。コードを批判的に読む訓練はこれから。

## 方針

**「概念理解はAI活用、構造理解は自分の手で」**

- 写経時間を削減し、読む・説明するという理解確認に置き換える
- AI活用自体も実務スキルとして体験させる
- コマごとにAIの使い方を変える（聞く→生成→攻撃側）

---

## タイムライン

### コマ1（50分）― DBを知る・触る

| 時間 | 内容 | スタイル |
|---|---|---|
| 5分 | 前回おさらい | 講義 |
| 10分 | DB概念（DBとは・テーブルとは） | AIに質問させて講師が補足 |
| 20分 | phpMyAdmin操作（DB・テーブル作成） | ハンズオン |
| 15分 | SQL INSERT / SELECT基礎 | AIで生成→読んで説明させる |

**AIの使い方：概念理解のために聞く**
- 「DBとテーブルって何が違うの？」とAIに質問させる
- 出てきた回答を全体で共有し、講師が補足・整理する

---

### コマ2（50分）― PHPとDBをつなぐ

| 時間 | 内容 | スタイル |
|---|---|---|
| 25分 | PDO（insert.php） | AIで雛形生成→講師解説→bindValueだけ自分で書く |
| 15分 | SELECT表示（select.php） | 同上 |
| 10分 | XSS対策（h()関数） | 講義＋写経（ここは手で書かせる） |

**AIの使い方：雛形生成→読んで理解**
- insert.phpの雛形をAIに生成させる
- 講師が各部分（PDO・prepare・bindValue）を解説
- bindValueの部分だけ自分で書かせる（手を動かす部分を残す）

**XSSだけ写経にする理由**
- h()関数はシンプルで短い
- 「自分で書いた防御コード」という意識を持たせたい

---

### コマ3（50分）― セキュリティ体感・まとめ

| 時間 | 内容 | スタイル |
|---|---|---|
| 10分 | SQLインジェクション体感 | AIに攻撃コード生成させて読む |
| 10分 | 宿題説明 | 講義 |
| 15分 | バッファ（追いつき・質疑応答） | 自由 |
| 15分 | プラスアルファ（早く終わった人向け） | 下記参照 |

**AIの使い方：攻撃側の視点で使う**
- プレースホルダなしのコードをAIに渡して「これに攻撃するSQL書いて」と依頼
- なぜ危ないかを読んで理解させる
- SQLインジェクションの詳細説明は次回以降

---

## プラスアルファ（早く終わった人向け）

優先度順：

### ① XSS攻撃を自分で試す（推奨）
1. select.phpの `h()` を外す
2. `<script>alert('XSS')</script>` を投稿
3. アラートが出ることを確認
4. `h()` を戻して防げることを確認

→ セキュリティを体で覚える。記憶に残りやすい。

### ② SELECTの応用をAIと遊ぶ
- 「登録日時の新しい順に3件だけ取得するSQL書いて」など要件をAIに渡す
- 生成されたSQLをphpMyAdminで実際に動かす
- ORDER BY・LIMIT・WHEREの体験

### ③ AIコードレビュー体験
- 自分が書いたinsert.phpをAIに渡して「改善点を教えて」と聞く
- AIのレビューを読んで、採用するかどうか自分で判断する
