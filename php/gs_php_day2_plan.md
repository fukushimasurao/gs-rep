# Day2 講師メモ（3時間版）

## PHP授業 全5日間の概要

| Day | テーマ | 学ぶこと |
|---|---|---|
| Day1 | PHP基礎 | 変数・配列・関数・ループ、フォーム操作、ファイル保存 |
| **Day2** | **DB入門** | **DB・テーブルの概念、SQL（INSERT/SELECT）、PHPからDB操作（PDO）** |
| Day3 | CRUD | 詳細表示・更新（UPDATE）・削除（DELETE）、コードの関数化 |
| Day4 | ログイン機能 | セッション管理、権限による処理の分岐、パスワードのハッシュ化 |
| Day5 | DBリレーション | テーブルの設計・正規化、JOINで複数テーブルを扱う |

## 前提

- 受講生：HTML/CSS/JS を約10時間学んだ初学者
- 授業時間：50分 × 3コマ（計150分）
- 宿題：ブックマークアプリ（授業内では扱わない）
- AI活用：調べもの程度には使える。コードを批判的に読む訓練はこれから。

## 方針

**「概念理解はAI活用、構造理解は自分の手で」**

- 写経時間を削減し、読む・説明するという理解確認に置き換える
- AI活用自体も実務スキルとして体験させる
- コマごとにAIの使い方を変える（聞く→生成→攻撃側）

**insert.php 最終フロー**

---

**insert.php（理解フェーズ）詳細**

1. 受講生がAIにスケルトンを穴埋めさせる
2. 授業資料の完成形コードと見比べる（「違う部分を探して」と問いかける）
3. 講師が4ブロックに沿ってコードを解説する（問いかけながら）
   - `1.DB接続` → PDOとは何か、try-catchの役割
   - `2.SQL作成` → prepareとプレースホルダの意味
   - `3.実行` → executeの役割
   - `4.実行後の処理` → 成功・失敗の分岐
4. AIのコードをinsert.phpにコピペして動作確認
5. 以下の2箇所を削除して自分で書き直す
   - prepare内のSQL文（phpMyAdminで書いたINSERT文の再利用）
   - bindValueの3行（Day3以降も毎回書くパターン）

**select.php 最終フロー**

1. AIに全部書かせる
2. 「insert.phpと4ブロックの構造が同じ」ことをサクッと確認
3. 講師がwhile+fetchを一言で説明（「Day1のforeachと同じで、データを1行ずつ取り出している」）
4. 動かして自分のデータが画面に出る瞬間を体験させる（ここが一番の見せ場）
5. 「なんでwhileなんだと思う？」と問いかけて終わり（テキストにも折りたたみで記載済み）

自分で書く部分はなし。「動いた！」の体験に全振りする。

## SQLインジェクション体感：授業で起きること（講師メモ）

`test'); DROP TABLE gs_an_table; --` をnameに入力した場合、PHPが組み立てるSQLはこうなる。

```sql
INSERT INTO gs_an_table (id, name, email, content, date) VALUES (NULL, 'test'); DROP TABLE gs_an_table; --', 'email', 'content', NOW())
```

最初のINSERT文が `VALUES (NULL, 'test')` で終わってしまい、5カラムに対して値が2つ → **「Column count doesn't match」エラーが発生する。**

**テーブルが消えない理由：**
PDOは1回の`query()`で複数SQL文を実行できない仕様のため、DROP TABLE部分はそもそも実行されない。

**ただし攻撃は成功している：**
- アプリがクラッシュしてエラー画面が表示される（サービス停止）
- エラー情報がそのまま画面に出る（情報漏洩）
- この2点だけでも十分なセキュリティインシデント

→ 「テーブルは消えなかったけど、攻撃は成功している」という点を授業で強調する。
